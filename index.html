Tree-based Data Structure

Why Tree Data Structure?

The time complexity of operations in linear data structure increases as the number of elements increases. Different tree data structures allow quicker and easier access to the data as it is non-linear.

Terminologies

  • Root : Topmost node of the tree
  • Parent of a node is the node connected above it.
  • Children of a node is the node connected below it.
  • Leaf : the node that have no children node.

    • Nodes are Sibling nodes if they have the same parent.

  • Edge : the connection between nodes.
  • Height of a node : is the number of edges from the node to the deepest leaf
  • Depth of a node : is the number of edges from the node to the root.
  • Depth of a tree : is the number of edges from the root node to the deepest ndoe
  • Degree of a node : is the total number of branches of that node.

    • Forest : is a collection of disjointed trees

Application of Trees

  1. To store information that naturally forms a hierarchy (ex.: file system).
  2. Make information easy to search, Self-balancing search trees like AVL and R-B trees are guaranteed to have time complexity of O(logn).
  3. Manipulate sorted lists of data in moderate time (quicker than array and slower than unordered linked lists.)
